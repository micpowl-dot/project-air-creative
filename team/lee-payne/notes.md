### Phase 2 — Templates & Systems (May 9–14)

Your job in Phase 2 is to build the frameworks that make Phase 3 fast. Every poster, badge, and piece of swag should feel like it came from the same campaign because you built the system it runs on.

**How to approach it:**

Build in this order — each one informs the next:

1. **Brand guidelines review** — as soon as Michael shares the logo and guidelines, internalize the colors, type, and creative direction before touching anything
2. **Poster visual system first** — this is the most visible deliverable and sets the tone for everything else. Lock the grid, hierarchy, and color application here.
3. **PPT template second** — shares DNA with the poster system but lives in its own world. 12 layouts, all built from the same type and color rules.
4. **Swag visual system last** — swag is smaller scale, so the system is simpler. Make sure it feels like the same campaign as the posters.

**Recommended tools:**

- **Adobe Illustrator** — poster and swag systems. Keep everything in one master file with artboards per layout variation.
- **Adobe InDesign** — PPT template. Build it here first for layout precision, then export to .potx. Alternatively build directly in PowerPoint if that's faster for the team.
- **Adobe Firefly** — use generative fill to explore background textures and atmospheric elements. Great for giving the poster system warmth and depth without stock imagery.
- **Fonts Ninja** (browser extension) — useful for identifying type you see in reference work that inspires you

### Phase 3 — Ambassador Posters ×17 (May 14–22)

You and Camille are splitting the 17 ambassador posters. Once Denise delivers the headshots and names, divide the list and run in parallel.

**How to approach it:**

1. **Build one poster template first** — one complete example with a real headshot, the ambassador's name, and title. Get Michael's approval on this before you produce the remaining 16.
2. **Batch production** — once the template is approved, drop each headshot in, update the name and title, export. This should be fast if the system is built right.
3. **Headshot consistency** — each photo will look different. Use Photoshop to normalize color temperature, contrast, and cropping across all 17 before dropping into the template.
4. **File naming** — name exports consistently: `D1-01_firstname-lastname_poster.pdf`. Makes Denise's trafficking job much easier.

**Recommended tools:**

- **Adobe Photoshop** — headshot retouching and background treatment
- **Adobe Illustrator** — poster layout production
- **Adobe Bridge** — batch export and file management across 17+ assets
- **Adobe Firefly** — background generation or extension if headshots have inconsistent backgrounds
